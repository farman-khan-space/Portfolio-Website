Welcome! This guide will walk you through creating new project and post files for your portfolio. It covers the required **front matter** (the settings at the top of the file) and provides a comprehensive cheatsheet for writing content in **Markdown**.

YAML Front Matter Explained

The "front matter" is the block of text at the very top of your markdown files, enclosed by `---`. This block contains key information (metadata) about your content. Below is a breakdown of each template.

Project Template (`project-template.md`)
KeyDescription & Example
title(Required) Main title of your project. Appears on the page and browser tab.
summary(Required) Short, one-sentence description for the main projects listing page.
date(Required) Publication date. Format: YYYY-MM-DD.
tags(Required) A list of tools, skills, or technologies used. These are used for filtering.
Example: `["Python", "Pandas", "Power BI"]`
descriptionA longer sentence for SEO (Google) and social media link previews. Be descriptive!
keywordsA list of keywords for search engines.
featuredImage(Required) Path to the image for the project summary card (800x600 recommended).
headerImage(Required) Path to the large banner image at the top of the project page (1920x1080 recommended).
ogImage(Required) Path to the image for social media sharing (1200x630 recommended).
Post Template (`post-template.md`)
KeyDescription & Example
title(Required) The title of your article.
summary(Required) Short description for the main blog listing page.
date(Required) Publication date. Format: YYYY-MM-DD.
readTime(Required) Estimated reading time. Example: "8 min read"
tags(Required) A list of topics related to the article.
Example: `["SQL", "Databases", "Tutorial"]`
The description, keywords, featuredImage, headerImage, and ogImage fields work the same as in the Project Template.

How to Write Beautiful Markdown Content

After the front matter, you write your content using Markdown. It's a simple way to format text that gets converted into HTML.

Headings & Table of Contents

Headings

Use hashtags (`#`) for headings. Your article title is Level 1 (`#`), so start your content with Level 2 (`##`).

## Problem Statement
### Data Cleaning
#### Handling Missing Values

Table of Contents (TOC)

Your website builder likely generates a TOC automatically from your headings. To link to a heading manually, convert it to lowercase and replace spaces with hyphens.

* [Go to the Problem Statement](#problem-statement)

Code Blocks & Snippets

Inline Code

For short snippets in a sentence, use single backticks (`` ` ``).

The `df.isnull().sum()` method is useful for finding missing values.

Result: The `df.isnull().sum()` method is useful for finding missing values.

Fenced Code Blocks (Recommended)

For multi-line code, use triple backticks (```` ``` ````) and specify the language for syntax highlighting.

Python Example:

```python
import pandas as pd

# Load the dataset from a CSV file
df = pd.read_csv('sales_data.csv')

# Display the first 5 rows to inspect the data
print(df.head())
```

SQL Example:

```sql
SELECT
    product_category,
    SUM(sales_amount) AS total_sales,
    AVG(customer_rating) AS avg_rating
FROM
    sales_records
WHERE
    sale_date >= '2025-01-01'
GROUP BY
    product_category
ORDER BY
    total_sales DESC;
```
Links & Images (In Content)

Links

Create links using the `[Text to display](URL)` format.

[View the dataset on Kaggle](https://www.kaggle.com/datasets/example)

Images

To embed an image in your content (like a chart or diagram), the syntax is `![Alt Text](Path to Image)`. This is different from the images in the front matter, which are used for the page layout.

![Sales Distribution by Product Category](/img/projects/market-trend/sales-chart.png "Sales Distribution Chart")
  • !: Tells Markdown this is an image.
  • [Alt Text]: Crucial for accessibility and SEO. Describe the image for screen readers and search engines (e.g., "Bar chart showing sales distribution across electronics, clothing, and groceries").
  • (Path to Image): The URL to your image. This should be a path relative to the root of your site. For a project called "market-trend", your images should be in a folder like `/img/projects/market-trend/`.
  • "Optional Title": Text that appears when a user hovers over the image.

Horizontal Rules

To create a thematic break or a horizontal line, use three or more hyphens, asterisks, or underscores on a new line.

Tables

Use pipes `|` and hyphens `-` to create tables. They are perfect for feature descriptions or result summaries.

| Feature         | Data Type | Description                              |
|-----------------|-----------|------------------------------------------|
| `age`           | integer   | The age of the customer in years.        |
| `income`        | float     | Annual income of the customer.           |
| `gender`        | object    | Gender of the customer ('Male'/'Female').|
