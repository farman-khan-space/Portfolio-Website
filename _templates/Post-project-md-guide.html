Welcome! This guide will walk you through creating new project and post files for your portfolio. It covers the required **front matter** (the settings at the top of the file) and provides a comprehensive cheatsheet for writing content in **Markdown**.

YAML Front Matter Explained

The "front matter" is the block of text at the very top of your markdown files, enclosed by `---`. This block contains key information (metadata) about your content. Below is a breakdown of each template.

Project Template (`project-template.md`)

Use this template for your data analysis case studies and projects.

Key Description & Example
title (Required) The main title of your project. Appears on the page and browser tab.
Example: "Customer Segmentation using K-Means Clustering"
summary (Required) A short, one-sentence description shown on the main projects listing page.
Example: "An analysis of retail data to identify distinct customer groups for targeted marketing."
date (Required) The publication date. Use the format YYYY-MM-DD.
Example: "2025-09-16"
tags (Required) A list of tools, skills, or technologies used. These are used for filtering.
Example: `["Python", "Pandas", "Scikit-learn", "Seaborn"]`
description A longer sentence for SEO (Google search results) and social media link previews. Be descriptive!
Example: "A detailed data analysis case study on customer segmentation using Python, Pandas, and Scikit-learn's K-Means algorithm to uncover actionable business insights."
keywords A list of keywords for search engines. Include relevant terms.
Example: `["customer segmentation", "k-means", "data analysis", "python"]`
featuredImage (Required) Path to the smaller image for the project summary card (800x600 recommended).
Example: `/img/projects/customer-segmentation/featured.png`
headerImage (Required) Path to the large banner image at the top of the project page (1920x1080 recommended).
Example: `/img/projects/customer-segmentation/header.jpg`
ogImage (Required) Path to the image used for social media sharing (e.g., LinkedIn, Twitter). 1200x630 is the standard "Open Graph" size.
Example: `/img/projects/customer-segmentation/social.png`
Post Template (`post-template.md`)

Use this template for blog posts, tutorials, and articles.

Key Description & Example
title (Required) The title of your article.
Example: "A Beginner's Guide to SQL Joins"
summary (Required) A short description for the main blog listing page.
Example: "Understand the differences between INNER, LEFT, RIGHT, and FULL OUTER joins with clear examples and visuals."
date (Required) The publication date. Use YYYY-MM-DD.
Example: "2025-09-18"
readTime (Required) An estimate of how long the article takes to read.
Example: "8 min read"
tags (Required) A list of topics related to the article.
Example: `["SQL", "Databases", "Data Analysis", "Tutorial"]`
The description, keywords, featuredImage, headerImage, and ogImage fields work exactly the same as in the Project Template.

How to Write Beautiful Markdown

After the front matter, you write your content using Markdown. It's a simple way to format text that gets converted into HTML.

Headings & Table of Contents

Headings

Use hashtags (`#`) to create headings. The number of hashtags corresponds to the heading level. Your article title is automatically Level 1 (`#`), so you should start your content with Level 2 headings (`##`).

## This is a Main Section (h2)
### This is a Sub-section (h3)
#### This is a Sub-sub-section (h4)

Table of Contents (TOC)

A TOC is usually generated automatically by your website builder (Eleventy) based on the headings you use. If you need to create one manually, you can use a list with links that point to heading IDs.

To create a link to a heading, use `[Link Text](#heading-text)`. The ID is typically the heading text in lowercase with spaces replaced by hyphens.

* [Link to Main Section](#this-is-a-main-section-h2)
* [Link to Sub-section](#this-is-a-sub-section-h3)

Code Blocks & Snippets

Inline Code

For short snippets, variable names, or file names within a sentence, wrap the text in backticks (`` ` ``).

To read a CSV file, use the `pd.read_csv()` function from the `pandas` library.

Result: To read a CSV file, use the `pd.read_csv()` function from the `pandas` library.

Fenced Code Blocks

For longer, multi-line code examples, use triple backticks (```` ``` ````). You can specify the language right after the opening backticks for syntax highlighting.

```python
import pandas as pd

df = pd.DataFrame({
    'name': ['Ali', 'Fatima', 'Ahmed'],
    'age': [28, 34, 29]
})

print(df.describe())
```

You can also use `sql`, `javascript`, `html`, `css`, etc.

Links & Images

Links

You can create links in two ways.

1. Inline Links (Most Common):

[This is a link to Google](https://www.google.com)

Result: This is a link to Google

2. Reference Links (Placeholders):

This is useful for keeping your paragraphs clean if you have many long URLs. Define the URL "placeholder" at the bottom of your file.

I use pandas for data manipulation. You can find its documentation [here][pandas-docs]. For visualization, I prefer [Seaborn][seaborn-home].

[pandas-docs]: [https://pandas.pydata.org/docs/](https://pandas.pydata.org/docs/)
[seaborn-home]: [https://seaborn.pydata.org/](https://seaborn.pydata.org/)

Images

The syntax is similar to links, but with a `!` at the beginning.

![This is the alt text for the image](/path/to/your/image.jpg "Optional Title Text")

Alt text is crucial for accessibility and SEO. It describes the image for screen readers and search engines.

Lists (Unordered, Ordered, and Task Lists)

Unordered List

Use `*`, `-`, or `+` for bullet points.

* Item 1
* Item 2
  * Sub-item 2.1
  * Sub-item 2.2

Ordered List

Use numbers followed by a period.

1. First step: Clean the data
2. Second step: Explore the data
3. Third step: Build the model

Task List

Create a checklist. The `[x]` marks an item as complete.

- [x] Import necessary libraries
- [x] Load the dataset
- [ ] Perform feature engineering
Blockquotes & Callouts

Use the `>` character to create a blockquote, which is great for quoting text or creating styled "callout" boxes for notes or warnings.

> "The goal is to turn data into information, and information into insight."
> - Carly Fiorina

Tables

Creating tables can be tedious, but they are great for presenting structured data. Use pipes `|` to separate columns and hyphens `-` to create the header row separator.

| `income`        | float     | Annual income of the customer.           |
| `gender`        | object    | Gender of the customer ('Male'/'Female').|
